# AdEarn

Полностью рабочее мини-приложение "в одном проекте": сервер сам отдаёт и
сайт (папка `public/index.html`), и API (базу данных, начисление наград,
вывод средств, живую ленту и топ). Один деплой — и работает и то, и то,
отдельный GitHub Pages не нужен.

## Что внутри

- **public/index.html** — сам вебапп (то, что видит пользователь в Telegram).
- **db.js** — вся база данных. Хранит всё в одном файле `data.json` рядом
  с кодом (создаётся сам при первом запуске). Работает на любой версии
  Node.js — специально выбрано без нативных зависимостей и без
  экспериментальных модулей, чтобы не ловить сюрпризы на разных хостингах.
- **telegramAuth.js** — проверка подлинности пользователя Telegram
  (проверяем подпись `initData`, чтобы никто не мог подставить чужой userId).
- **server.js** — API на Express + раздача `public/index.html`.

## Запуск

```bash
npm install
cp .env.example .env
# впиши в .env токен своего бота (взять у @BotFather -> /mybots -> API Token)
npm start
```

Сервер поднимется на `http://localhost:3000`.

База хранится в файле `data.json` рядом с кодом — простой текстовый файл,
который можно открыть и посмотреть глазами при необходимости. Работает
на любой версии Node.js (проверено на 20.x и 22.x). Если пользователей
станет много тысяч и понадобится настоящая БД — можно перейти на
PostgreSQL (`pg`), структура функций в `db.js` останется той же.

## Как это работает изнутри

Фронтенд (`public/index.html`) теперь сам обращается к API того же сервера
(`fetch("/api/...")` без домена — потому что оба живут по одному адресу):

1. При старте — отправляет `Telegram.WebApp.initData` на `POST /api/auth`.
2. При клике «Смотреть рекламу» — вызывает `AdController.show()`, а после
   этого **не начисляет награду сам**, а несколько раз перепроверяет баланс
   через `GET /api/me`, ожидая, пока AdsGram подтвердит просмотр на сервере
   (см. следующий раздел).
3. Вкладки «Выводы» и «Топы» — тянут данные из `GET /api/live-withdrawals`
   и `GET /api/leaderboard`. Пока в базе нет ни одного пользователя/вывода,
   там честно будет пусто ("пока никто не выводил") — это ожидаемо для
   свежего проекта, не баг.
4. Кнопка «Отправить заявку» — шлёт `POST /api/withdraw`.

## Настройка AdsGram Reward Url (важно для защиты от накруток)

В личном кабинете AdsGram (partner.adsgram.ai) у своего блока укажи:

```
Reward Url: https://твой-домен.ru/reward?userid=[userId]
```

AdsGram сам подставит вместо `[userId]` Telegram ID пользователя и вызовет
этот адрес, когда пользователь реально досмотрит рекламу до конца. Именно
это событие, а не клиентский `.then()` в браузере, должно быть источником
истины для начисления денег — клиентский вызов легко подделать.

## Эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/auth` | Вход по Telegram initData, создаёт пользователя |
| GET  | `/reward?userid=` | Callback от AdsGram, начисляет награду |
| GET  | `/api/me?userid=` | Баланс и статистика пользователя |
| POST | `/api/withdraw` | Создать заявку на вывод |
| GET  | `/api/my-withdrawals?userid=` | История выводов пользователя |
| GET  | `/api/live-withdrawals` | Живая лента (реальные данные) |
| GET  | `/api/leaderboard` | Топ по заработку |

## Дальше по-хорошему нужно

- Админку/скрипт для смены `status` заявок на вывод (`pending -> approved`)

- HTTPS и нормальный домен (Telegram Mini App и AdsGram callback требуют https).
- Rate limiting на `/api/withdraw` и `/reward`, чтобы не заспамили.
